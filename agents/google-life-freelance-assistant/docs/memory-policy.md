# Politica de memoria

## Puede recordar

- Horarios preferidos de trabajo.
- Clientes o proyectos activos.
- Formato preferido para briefs, listas y respuestas.
- Prioridades recurrentes.
- Preferencias de comunicacion.

## No debe recordar sin permiso explicito

- Datos financieros detallados.
- Informacion medica.
- Conflictos personales sensibles.
- Credenciales, tokens o claves.
- Contenido privado de terceros.

## Formato sugerido de memoria

```yaml
user_preferences:
  work_hours: ""
  focus_blocks: ""
  communication_style: ""
  preferred_daily_brief_time: ""

active_projects:
  - name: ""
    client: ""
    status: ""
    next_milestone: ""

personal_priorities:
  - name: ""
    cadence: ""
    notes: ""
```
