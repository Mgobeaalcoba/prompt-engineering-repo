# Guia de contribucion

Gracias por querer mejorar esta biblioteca de prompts.

## Antes de contribuir

- Revisa si ya existe un prompt similar.
- Usa la plantilla de `templates/prompt-template.md`.
- Escribe ejemplos claros, especificos y reutilizables.
- Evita incluir datos personales, secretos, claves de API o informacion sensible.

## Como agregar un prompt

1. Elegi la carpeta de tecnica mas adecuada dentro de `prompts/`.
2. Crea un archivo Markdown con nombre descriptivo en kebab-case.
3. Completa todas las secciones de la plantilla.
4. Agrega variables editables entre corchetes, por ejemplo `[producto]`.
5. Inclui criterios de calidad para evaluar la salida.

## Criterios de aceptacion

Un buen prompt debe:

- Tener un objetivo claro.
- Reducir ambiguedades.
- Definir formato de salida.
- Incluir restricciones importantes.
- Ser facil de adaptar.
- Evitar pedir al modelo que invente datos no disponibles.

## Estilo

- Idioma principal: espanol.
- Tono: practico, directo y reusable.
- Formato: Markdown.
- Preferir listas, tablas y salidas estructuradas cuando ayuden a comparar resultados.
