# Guia para mantener la biblioteca

## Flujo recomendado

1. Crear o adaptar un prompt.
2. Probarlo con al menos dos entradas distintas.
3. Ajustar instrucciones, variables y formato.
4. Documentar limitaciones.
5. Agregarlo a la carpeta correspondiente.

## Taxonomia sugerida

- `analisis`: prompts para diagnosticar, comparar o resumir.
- `creacion`: prompts para generar textos, ideas, planes o codigo.
- `evaluacion`: prompts para revisar calidad, errores o riesgos.
- `transformacion`: prompts para convertir formatos o reescribir contenido.
- `operacion`: prompts para procesos recurrentes de trabajo.

## Checklist de revision

- El prompt tiene un proposito claro.
- El output esperado esta definido.
- Las variables editables son faciles de identificar.
- No hay datos sensibles.
- No promete resultados imposibles.
- Incluye criterios de calidad.
