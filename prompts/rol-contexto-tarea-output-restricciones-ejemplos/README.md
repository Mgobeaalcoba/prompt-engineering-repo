# ROL, Contexto, Tarea, Output Esperado, Restricciones y Ejemplos

Este patron organiza prompts robustos para tareas recurrentes. Es ideal cuando varias personas necesitan usar o mantener el mismo prompt.

## Estructura recomendada

- **ROL**: quien debe actuar el modelo.
- **Contexto**: informacion de fondo necesaria.
- **Tarea**: accion concreta.
- **Output esperado**: formato de respuesta.
- **Restricciones**: limites, tono, fuentes, longitud o reglas.
- **Ejemplos**: muestras de entrada y salida.

## Ejemplos incluidos

- `analista-producto-feedback.md`
- `redactor-linkedin-b2b.md`
- `consultor-estrategia.md`
- `revisor-documentacion-tecnica.md`
- `mentor-aprendizaje.md`
